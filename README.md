# Wayfare

> *"The cost of a journey."*

A slow-travel companion for tracking what a temporary life abroad actually costs — receipts, places, memories, and spend, mapped day by day. Sibling to [The Slow Travel Planner](https://theslowtravelplanner.com).

## What it does

- **Spend tracking** — transactions, accounts, budgets, goals, subscriptions
- **Trip memory** — drop a receipt photo → AI extracts vendor/items/total → geocoded pin on a map
- **Daily story** — day-by-day timeline with photos, weather, and items per stop
- **Insights** — DCC fee detection, peak spend hours, tip patterns, top vendors
- **Bank linking** — Plaid (US) + GoCardless (EU/UK)

## Architecture

```
wayfare/
├── artifacts/
│   ├── api-server/        Express 5 API (TypeScript, esbuild)
│   ├── budget-web/        React + Vite web app  →  /

├── lib/
│   ├── db/                Drizzle ORM schema + config
│   ├── api-spec/          OpenAPI spec (orval codegen source)

│   └── api-client-react/  Generated React Query hooks
```

**Stack:** pnpm workspaces · Node 24 · TypeScript 5.9 · Express 5 · PostgreSQL · Drizzle ORM · React 19 · Vite 7 · Expo SDK 53 · Tailwind CSS v4 · shadcn/ui · Lora + Inter type

## Quick start (fresh clone)

### Prerequisites

- Node 24+
- pnpm 10+
- A PostgreSQL database (connection string ready)

### 1 — Install

```bash
pnpm install
```

### 2 — Configure environment

```bash
cp .env.example .env
# Edit .env — at minimum set DATABASE_URL
```

### 3 — Push database schema

```bash
pnpm --filter @workspace/db run push
```

### 4 — Seed demo data (optional)

```bash
pnpm --filter @workspace/scripts run seed
```

### 5 — Run everything

```bash
# Terminal 1 — API server (port 5000)
pnpm --filter @workspace/api-server run dev

# Terminal 2 — Web app (port configured via PORT env)
pnpm --filter @workspace/budget-web run dev

# Terminal 3 — Mobile (Expo)
pnpm --filter @workspace/budget-mobile run dev
```

## Scripts

| Command | What it does |
|---|---|
| `pnpm install` | Install all workspace dependencies |
| `pnpm run typecheck` | Full typecheck across all packages |
| `pnpm run build` | Typecheck + build all packages |

## Database

Schema lives in `lib/db/src/schema/`. Tables:

| Table | Purpose |
|---|---|
| `accounts` | Bank/cash/investment accounts |
| `categories` | Income/expense categories |
| `transactions` | All transactions (with holiday memory fields) |
| `budgets` | Monthly spending limits per category |
| `goals` | Savings goals with progress |
| `subscriptions` | Recurring subscriptions |
| `trackers` | Holiday/trip trackers |

**DB setup:** `pnpm --filter @workspace/db run push` (uses `drizzle-kit push` — applies schema directly, suitable for dev/Replit).

No migration history is generated by default. For production, switch to `drizzle-kit generate` + `drizzle-kit migrate` and commit the migrations folder.

## API codegen

The OpenAPI spec is at `lib/api-spec/openapi.yaml`. Client hooks and Zod validators are generated by Orval:

```bash
pnpm --filter @workspace/api-spec run codegen
```

Generated files (committed, do not edit manually):
- `lib/api-client-react/src/generated/` — React Query hooks
- `lib/api-zod/src/generated/` — Zod schemas

## Environment variables

See `.env.example` for the full reference with descriptions. Required at minimum:

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | ✅ | Postgres connection string |
| `SESSION_SECRET` | ✅ | Express session signing key |
| `OPENAI_API_KEY` | For AI receipt scan | Via Replit OpenAI integration |
| `PLAID_CLIENT_ID` / `PLAID_SECRET` | For bank linking | Plaid dashboard |
| `GOCARDLESS_SECRET_ID` / `GOCARDLESS_SECRET_KEY` | For bank linking | GoCardless dashboard |
| `DEFAULT_OBJECT_STORAGE_BUCKET_ID` | For photo uploads | Replit App Storage |
